## CRM — требования маркетинга

- **Цель**: убрать отдельные таблицы и вести весь маркетинговый и клиентский учёт в одной CRM.

### 1. Работа с клиентами и лидами

- [x] **Доп. поля при создании клиента**: источник (откуда пришёл), где написал (канал: TG, WhatsApp, Insta и т.п.), любые маркетинговые примечания.
- [ ] **Связка с заказом**: при первом заказе эти данные привязываются к номеру телефона / Telegram.
- [x] **Статусы**:
    - клиенты без заказов автоматически помечаются как «первичные / неактивные заявки»;
    - реализован удобный список таких клиентов для дальнейшей обработки (виджет в дашборде).
- [x] **Поле «пароль»**: убрать из обязательных при создании клиента в интерфейсе оператора (ему это поле не нужно).

### 2. Дашборд и аналитика

- [x] **План выручки**:
    - возможность задать план выручки на месяц;
    - прогресс-бар выполнения плана (факт / план) на дашборде.
- [x] **Показатели на дашборде**:
    - текущая выручка;
    - прибыль;
    - количество первичных заявок / клиентов без заказа (есть отдельный виджет-список таких клиентов);
    - количество текущих (активных) клиентов (со списком по клику);
    - средний чек.
- [x] **Напоминания**:
    - список клиентов, по которым через 14 дней после заказа нужно выйти на связь (виджет заказов, выданных более 2 недель назад, с переходом в клиента).

### 3. Воронка / путь клиента в CRM

- [ ] **Колонки воронки с количественными показателями**:
    - первичное обращение (сколько обращений всего);
    - заявка (сколько из обращений перешло в заявки);
    - первичная покупка (сколько из обращений дошло до первой покупки);
    - повторное обращение (сколько клиентов вернулось повторно);
    - повторная покупка (сколько из повторных обращений завершилось покупкой).
- [ ] **Задача**: видеть путь клиента от первого касания до повторных покупок в одной системе, без сторонних таблиц.
