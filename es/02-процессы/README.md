# 02 — Ценностные потоки (процессы)

## Опросник (группа 3) — ✅ завершён

### P-01: Заказ на заточку / ремонт (основной)

| Поле | Значение |
|------|----------|
| Триггер | Клиент обращается (телефон, визит, соцсети, сайт) |
| Типы | Заточка; ремонт; **в одном заказе может быть и то и другое**; срочный ремонт; гарантийный (привязка к предыдущему заказу) |
| Участники | Клиент, менеджер, мастер, курьер (вне системы как роль) |

**Happy path:**

1. Клиент обращается (канал любой).
2. **С сайта:** заявка → менеджер обрабатывает → создаёт заказ вручную.
3. **Обычно:** менеджер сразу создаёт заказ в системе.
4. Заказ со статусом **«новый»** → менеджер **назначает мастера** → мастер видит в POS.
5. Мастер берёт в работу → **«в работе»**.
6. При необходимости → **«ожидание запчастей»**; материалы на заказ добавляет **только менеджер**.
7. Мастер указывает выполненные работы (**наименование без цены**).
8. **Менеджер** назначает цены за работы; итог = работы + материалы.
9. Мастер завершает → **«готов»**.
10. Клиент забирает (сам или курьер) → менеджер переводит в **«выдан»**.
11. Клиент может оставить отзыв (после выдачи).

**Статусы:** `new` → `in_work` → `waiting_parts`? → `ready` → `issued` | `cancelled`

**Отмена:** менеджер, **только из статуса `new`** (уточнено в группе 5).

**Выдача (`ready` → `issued`):** только менеджер.

### P-02: Доставка (опция заказа)

| Шаг | Варианты |
|-----|----------|
| Сдача | Клиент приносит сам **или** курьер забирает |
| Возврат | Клиент забирает сам **или** курьер привозит |
| Расписание | Доставка по расписанию (окно, напр. 13–17) |

_Логистика курьера в системе не моделируется (роли курьера нет)._

### P-03: Гарантийный ремонт

- Отдельный сценарий с **привязкой к предыдущему заказу**.
- Оплата клиентом не требуется (гарантия).

### P-03b: Учёт оборудования

- Реестр аппаратов: бренд, модель, **серийник**, история ремонтов.
- Актуально для ремонта; мастер ищет по серийнику в POS.

### P-04: Документы

| Момент | Документ |
|--------|----------|
| При создании заказа | Квитанция / приём (уточнить шаблон) |
| При выдаче | Акт приёма-передачи |

### P-05: Склад

- Приход и номенклатура — **менеджер** в админке.
- Списание на заказ — **вручную** отдельной операцией менеджером (не автоматом в MVP).

### P-06: Личный кабинет клиента

| Функция | MVP |
|---------|-----|
| Профиль (имя, телефон, адрес доставки и т.д.) | ✅ |
| Активные заказы / статусы | ✅ |
| История заказов | ✅ |
| Отзыв после выдачи | ✅ |
| Бонусы, документы, повтор заказа | ❌ не в MVP |

**Связка гостя с аккаунтом:** менеджер вручную связывает заказы гостя с зарегистрированным клиентом.

**Отображение в ЛК:** клиент **не видит** статус цеха (in_work, ready…); только разделы «активные» / «история».

### P-07: Уведомления

- **Telegram — не в MVP** (во фронте есть UI — артефакт, не источник истины).
