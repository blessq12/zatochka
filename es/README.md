## Итог ES-сессии (кратко)

### Продукт

Мастерская в Томске: заточка + ремонт для частных мастеров beauty. Доставка — флаг на заказе. Один филиал.

### Три канала

| Канал | Кто | Назначение |
|-------|-----|------------|
| Публичный API + SPA | Клиент | Bootstrap, SubmitSiteLead, ЛК, отзывы |
| POS API | Мастер | Воронка (только свои заказы), работы, заметки |
| Filament `/cp` | Менеджер | Заказы, назначение мастера, цены, склад, выдача |

### Ядро домена

**Order** — центральный агрегат: `Tool[]`, `Work[]`, `OrderMaterial[]`, `equipment_id`, snapshot клиента.

**Поток:** Lead (сайт) → менеджер `CreateOrder` → `AssignMaster` → мастер `TakeOrderToWork` → … → `ready` → менеджер `IssueOrder`.

### Технические решения

- Монолит Laravel, 6 BC, `app/Domain/{BcName}/`
- Domain events синхронно (Laravel events)
- Чтение: прямой Eloquent, новый REST API
- PDF: DomPDF/Snappy, без хранения файлов в MVP
- Telegram, оплата, 1С — не в MVP

### Главные расхождения с текущим фронтом

1. Заявка с сайта ≠ заказ (Lead → менеджер)
2. Статусы в ЛК клиенту не показываем
3. Мастер видит только назначенные заказы
4. Пересчёт цены — явная команда менеджера
5. «Завершённые» в POS = только `ready`
